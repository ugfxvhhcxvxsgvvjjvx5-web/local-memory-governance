# EXECUTABLE_DEPENDENCY_RISK

> 用人话说明：为什么 exe / 解压目录 / 配置文件 / 便携软件 / trainer / 启动器不能像普通文档那样移动归位

---

## 为什么普通文档可以随便移，exe 不行？

一个 .md 文件，你从桌面移到 D:\Archive，它照样能打开，内容不会变。

但一个 exe 不一样。它可能：
* 启动时在自己"身边"找 dll
* 在自己"身边"找配置文件
* 在自己"身边"读取存档或数据库
* 被别的程序通过硬编码路径调用
* 被桌面快捷方式通过绝对路径指向

你一移动它，这些关系**全断了**。

---

## 风险 1：相对路径依赖

很多程序启动时会在 `当前目录` 或 `exe所在目录` 下寻找依赖文件。

```
v2rayN-windows-64/
  ├── v2rayN.exe        ← 你以为只要移这个
  ├── v2rayCore.dll     ← 但它启动时找这个
  ├── config.json       ← 和这个
  ├── guiConfigs/       ← 和这里的订阅数据
  └── logs/             ← 和运行日志
```

你把 v2rayN.exe 单独移到别处 → 启动就报错。
你把整个目录移到别处 → 快捷方式找不到了。

---

## 风险 2：邻居文件依赖

很多 exe 不是"单文件程序"，它和旁边的文件是**一个整体**。

典型例子：
* `FlingTrainer.exe` 旁边的 `FlingTrainer.dll` 和训练器数据
* 解压后的便携软件目录里的全部文件
* 游戏修改器旁边的注入 dll

**移一个就等于废了整套**。

---

## 风险 3：快捷方式失效

桌面快捷方式 (.lnk) 里存的是**绝对路径**：

```
快捷方式 "风灵月影修改器.lnk"
  → 目标: E:\download\FLiNGTrainer\FlingTrainer.exe
```

你把 `E:\download\FLiNGTrainer\` 移到 `D:\software\FLiNGTrainer\`：
* 快捷方式指向旧路径 → 点击报错"找不到目标"
* 你可能不知道有几个快捷方式指向这个目录
* 修复漏掉一个就多一个坏的快捷方式

---

## 风险 4：脚本硬编码路径

HOME 根目录的脚本可能被其他脚本或计划任务通过**完整路径**调用：

```bash
# 某个计划任务或开机脚本里写的：
wscript.exe "C:\Users\Administrator\start-gmn-proxy.vbs"
```

```vbs
' start-gmn-proxy.vbs 里写的：
Set shell = CreateObject("WScript.Shell")
shell.Run "node C:\Users\Administrator\gmn-proxy.mjs", 0
```

你把 `gmn-proxy.mjs` 移到 `D:\AgentTeam-Workspace\Dev\tools\`：
* 计划任务照旧调 vbs → vbs 找不到 mjs → 代理起不来 → 网络断了

**这种错你可能几天后才发现**。

---

## 风险 5：配置文件和主程序分离

有些程序的配置文件和主程序在不同目录，但通过路径关联：

* `.mcp.json` 写在 HOME，但被 Antigravity 从 HOME 读取
* `AGENTS.md` 写在 HOME，但被多个 AI 工具从 HOME 读取
* `package.json` 写在项目根，但 node_modules 在同目录

你移了配置文件但没移主程序（或反过来）→ 程序行为异常但不报明显错误。

---

## 风险 6：便携软件目录整体运行

便携软件 = 不安装、直接从目录运行的软件。

```
v2rayN-windows-64/
  └── v2rayN.exe → 直接运行，不注册系统服务
```

便携软件的运行依赖**目录完整性**：
* 所有文件必须在同一个目录
* 配置存在当前目录（不在 AppData）
* 日志写在当前目录
* 下载的资源存在当前目录

移动便携软件 = 移动整个目录。但如果有快捷方式指向旧位置，或者其他脚本引用旧路径，移了也是白移。

---

## 风险 7：游戏辅助和本体路径绑定

游戏修改器（风灵月影 / WeMod / Trainer）通常需要：

1. 知道游戏安装在哪（自动检测或手动指定）
2. 在游戏进程启动后注入 dll
3. 某些 trainer 会在自身目录下存储配置

如果你把修改器移到别处：
* 修改器可能无法找到游戏
* 之前配置好的游戏路径失效
* 注入 dll 路径可能断裂

---

## 总结

| 你以为的操作 | 实际后果 |
|-------------|---------|
| "就移个 exe 而已" | dll 找不到，程序崩溃 |
| "把解压目录归个位" | 快捷方式全废，代理断了 |
| "把脚本移到 tools 目录" | 计划任务找不到，服务起不来 |
| "配置文件也归位" | AI 工具读不到配置，行为异常 |
| "修改器放到 game 目录" | 找不到游戏本体，注入失败 |
| "安装包用完就删" | 如果安装不完整就没法重装了 |

**结论：先审计依赖，再决定能不能动。动之前确认快捷方式、脚本引用、配置关联全部处理好。**

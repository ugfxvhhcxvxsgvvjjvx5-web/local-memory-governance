## 核心原则

**exe 文件不等于普通文件。很多 exe、解压目录、配置文件、脚本、dll、mod、启动器都可能存在路径依赖，不能按普通归位逻辑处理。**

之前任何涉及 exe / 解压目录 / 配置文件 / 高风险运行资产迁移的想法，**全部暂停**，先审计再说。

---

## 高风险运行资产定义

以下内容一律视为**高风险运行资产**，在没有完成依赖审计前：

**不移动、不删除、不归档、不改名**

### 可执行文件
* `.exe` — 可执行程序
* `.msi` — 安装包
* `.bat` — 批处理脚本
* `.cmd` — 命令脚本
* `.ps1` — PowerShell 脚本
* `.vbs` — VBScript
* `.ahk` — AutoHotkey 脚本
* `.sh` — Shell 脚本
* `.dll` — 动态链接库

### 配置文件
* `.ini` — Windows 配置
* `.json` 配置文件（如 .mcp.json / .claude.json / package.json）
* `.cfg` / `.conf` — 通用配置

### 目录类型
* 解压后的软件目录（如 v2rayN-windows-64/）
* 便携软件目录
* 游戏目录
* 修改器 / trainer / mod 目录
* 启动器目录
* 工具目录（内含 exe + dll + config 的整体运行环境）

### 数据类型
* 数据库文件（.db / .sqlite / .mdb）
* Obsidian Vault 目录
* 记忆系统目录

### 引用类文件
* 可能被快捷方式（.lnk / .url）指向的文件
* 可能被脚本或计划任务调用的文件
* 可能被其他程序硬编码路径引用的文件

---

## 审计流程（在允许移动之前必须完成）

1. **确认类型**：是安装器还是运行体？
2. **确认依赖**：旁边是否有 dll / json / ini / data / config / logs / assets？
3. **确认引用**：是否有快捷方式指向它？是否被脚本调用？
4. **确认状态**：是否仍在使用？还是已废弃？
5. **决策**：只有当以上 4 项全部确认后，才能决定是移动、归档还是删除
